CHAPTER · WHY MODELING

피지컬 컴퓨팅인데, 왜 3D 모델링부터?

많은 학생들이 처음에 이렇게 묻습니다. "코딩 수업인데 왜 블렌더부터 켜요?"

이유는 단순합니다. 피지컬 컴퓨팅은 "물리적인" 컴퓨팅이거든요. 코드는 가상 세계에서 끝나지 않고, 실제로 회전하는 바퀴와 감지하는 센서, 그리고 그것들이 매달려 있을 "몸체"로 결과를 냅니다. 그 몸체가 엉성하면 아무리 코드가 완벽해도 차는 비뚤어지게 굴러갑니다.

또 자율주행에서는 센서의 위치 = 알고리즘의 성능입니다. IR 센서를 너무 높이 달면 라인을 못 보고, 카메라가 흔들리면 OpenCV로 차선을 못 찾습니다. "하드웨어의 형태가 곧 소프트웨어의 제약"이라는 사실을 실제로 깨닫는 시간입니다.

그리고 한 가지 더 — 자율주행은 여러 분야가 합쳐진 종합 학문입니다. 모델링을 배우면 학생들이 "내가 만든 차"라는 애착을 갖게 되고, 이게 끝까지 끌고 가는 가장 큰 동력이 됩니다.

1

설계가 먼저, 회로는 그 다음

모터 두 개의 위치, 센서 사이의 간격, 배터리 무게 중심 — 모델링 단계에서 결정된 게 그대로 주행 성능을 좌우합니다.

2

3D 프린팅과 직결

오늘 만든 STL 파일이 다음 차시의 진짜 부품이 됩니다. "그린 게 손에 잡힌다"는 경험은 코딩과는 또 다른 동기를 줍니다.

3

공간 감각 = 디버깅 능력

센서가 어디를 보고 있는지 3D로 상상할 수 있어야, 차가 왜 라인을 놓쳤는지 코드만 보고도 추론할 수 있게 됩니다.

4

"내 차"라는 애착

이건 기술적이지는 않지만 가장 중요합니다. 직접 디자인한 차여야 끝까지 만들고 싶어집니다.

🧊
오늘의 모델링 목표: Blender 기본 조작(G/R/S, Extrude, Boolean) 익히기 → 모터·센서·아두이노가 모두 들어가는 섀시(차체) 설계 → STL 내보내기. 정확한 치수보다 "왜 이 형태여야 하는지"를 이해하는 것이 목표입니다.

🎬 블렌더 단축키 — 7개만 외우면 끝

블렌더는 마우스보다 키보드 단축키로 작업합니다. 처음엔 막막해도 아래 7개만 익히면 오늘 수업의 90%는 해결됩니다. 각 상자는 위에서 아래로 순서대로 배치되어 있고, 오른쪽 작은 화면이 실제 동작 모습이에요.

G

이동 · Grab

오브젝트를 잡고 끌어 이동합니다. G 누른 뒤 X·Y·Z를 입력하면 그 축으로만 고정 이동돼요.

R

회전 · Rotate

오브젝트를 돌립니다. R Z 90 처럼 숫자를 입력하면 정확한 각도로 회전.

S

크기 · Scale

크기를 키우거나 줄입니다. S X 2 처럼 축과 배수 입력 가능.

E

돌출 · Extrude

면을 뽑아내 형태를 확장합니다. (Edit 모드에서만 동작) 차체 받침대를 만들 때 핵심.

Tab

모드 전환 · Object ↔ Edit

오브젝트 전체를 다룰지(Object), 점·선·면을 다룰지(Edit) 모드를 전환합니다.

Shift + A

추가 · Add Menu

큐브·실린더·구 같은 기본 도형을 추가합니다. 모델링은 거의 여기서 시작.

X

삭제 · Delete

선택한 오브젝트/점/선/면을 지웁니다. 메뉴가 뜨면 무엇을 지울지 선택.

① 디지털 출력 — LED 깜빡이기 (Blink)

Blink.ino 
void setup() {
  pinMode(13, OUTPUT);
}

void loop() {
  digitalWrite(13, HIGH);
  delay(1000);
  digitalWrite(13, LOW);
  delay(1000);
}

핵심 함수

pinMode(핀, 모드) INPUT / OUTPUT 설정
digitalWrite(핀, 값) HIGH(5V) / LOW(0V)
delay(ms) 1000 = 1초
회로: 13번 핀 → 저항(220Ω) → LED(+) → GND
UNO 보드의 내장 LED로 바로 테스트도 가능!

② 디지털 입력 — 버튼 읽기

INPUT_PULLUP 모드를 쓰면 안 눌렀을 때 자동으로 HIGH가 됩니다. 버튼을 누르면 GND와 연결되어 LOW로 떨어져요. "눌렀을 때 LOW"라는 게 처음엔 헷갈리지만 익숙해지면 가장 편한 방식입니다.

digitalRead(핀) HIGH 또는 LOW 반환
⚠️
채터링(chattering): 버튼은 한 번 눌러도 짧은 시간 동안 HIGH/LOW가 여러 번 튑니다. 정밀한 입력엔 디바운싱이 필요해요.
Button.ino 
const int BTN = 2;
const int LED = 13;

void setup() {
  pinMode(BTN, INPUT_PULLUP);
  pinMode(LED, OUTPUT);
}

void loop() {
  if (digitalRead(BTN) == LOW) {
    digitalWrite(LED, HIGH);
  } else {
    digitalWrite(LED, LOW);
  }
}

③ 시리얼 통신 — 디버깅의 생명줄

Serial.ino 
void setup() {
  Serial.begin(9600);   // 속도 9600bps
}

void loop() {
  int val = analogRead(A0);
  Serial.print("값: ");
  Serial.println(val);
  delay(200);
}

하드웨어 코딩에서 "눈" 역할을 하는 게 시리얼 통신입니다. 센서 값이 진짜로 잘 들어오는지, 코드가 어디서 멈췄는지 — 전부 시리얼 모니터로 확인합니다. 앞으로 모든 실습에서 시리얼은 항상 켜두세요.

Serial.begin(9600) 통신 시작
Serial.println(값) 출력 + 줄바꿈
💡
IDE 우측 상단 돋보기 = 시리얼 모니터. 속도(보레이트)가 코드와 같아야 글자가 안 깨집니다.

① 라인트레이서란? — 설명과 원리

라인트레이서는 이름 그대로 "선(line)을 추적(trace)하는" 차입니다. 바닥에 그려진 검은 선을 두 개의 IR 센서가 감시하면서, 선이 왼쪽으로 치우치면 왼쪽으로 꺾고, 오른쪽으로 치우치면 오른쪽으로 꺾어 "항상 두 센서 사이에 선이 있도록" 차를 조정합니다. 복잡해 보이지만 결국 센서 읽기 → 조건문 → 모터 명령의 단순한 반복이에요.

차의 앞쪽 바닥에 IR 적외선 센서 2개를 답니다. 적외선을 쏘고 반사되어 돌아오는 빛의 양을 측정해서, 바닥이 흰색(반사 많음)인지 검은색(반사 적음)인지 판단해요. 두 센서의 조합에 따라 다음 4가지 상태가 만들어집니다.

L=0 · R=0

직진 ⬆️

둘 다 흰 바닥

L=1 · R=0

좌회전 ↩️

왼쪽이 선 감지

L=0 · R=1

우회전 ↪️

오른쪽이 선 감지

L=1 · R=1

정지 🛑

교차로/도착점

이 4가지 상태 = if-else 4줄. 자율주행이라는 거창한 단어가 결국 "센서 보고 조건문 분기"라는 걸 직접 느껴봅시다.
🚸
여기서 한 단계 더 — 충돌 방지. 진짜 자율주행차는 라인만 보지 않습니다. 앞에 보행자나 장애물이 있으면 무조건 멈춰야 해요. 그래서 우리는 차 앞쪽에 초음파 센서(HC-SR04)를 하나 더 달아서, 일정 거리 이내(15cm)에 무언가 있으면 라인 추적을 잠시 중단하고 정지하는 안전 기능을 추가합니다. "안전이 라인보다 우선"이라는 자율주행의 핵심 원칙이에요.

② 그런데, 왜 라인트레이서일까?

QUESTION
"자율주행 자동차를 배운다면서, 왜 처음에 라인만 따라가는 차를 만들어요?"
좋은 질문입니다. 답은 두 가지예요.

① 라인트레이서 = 차선 검출의 가장 첫 단계. 진짜 자율주행차도 결국은 "도로의 차선을 보고 그 안에서 달린다"는 일을 합니다. 라인트레이서는 그 일을 가장 기본적인 센서(IR)와 가장 단순한 알고리즘(if 분기)으로 해보는 것이에요. 내일 OpenCV로 차선 검출을 배울 때, 오늘 만든 이 차가 "아, 똑같은 일을 카메라로 하는 거구나" 하고 확 연결됩니다.

② 아두이노 수업의 개념 복습. 앞에서 배운 setup/loop, digitalRead, digitalWrite, if 분기, 시리얼 — 이 모든 것이 라인트레이서 한 대 안에 다 들어갑니다. 새로운 걸 배우는 게 아니라, 방금 배운 것으로 진짜 동작하는 무언가를 완성하는 시간이에요.

그래서 오늘 만들 이 작은 차 한 대가, 사실은 자율주행이라는 거대한 주제로 들어가는 가장 정확한 첫 발자국입니다.

③ 만드는 과정

부품을 챙기고 → 핀을 연결하고 → 차로 조립하면 끝. 코드 작성 전에 물리적인 준비를 먼저 마쳐야 코드를 짤 때 헤매지 않습니다.

④ 코드 살펴보기

한 번에 전체 코드를 보면 길어 보이지만, 사실은 "핀 정의 → 모터 동작 함수 → 거리 측정 함수 → 메인 loop" 네 덩어리로 나눠집니다. 한 덩어리씩 보면 굉장히 단순해요. 각 부분이 무엇을 하는지 먼저 이해하고, 맨 마지막에 합쳐서 전체를 봅시다.

STEP 4-1 · 핀 정의와 setup()

① 핀 정의 + setup 
const int IR_L = 2;
const int IR_R = 3;
const int TRIG = 9;   // 초음파 발사
const int ECHO = 10;  // 초음파 수신
const int IN1 = 5;   // 왼쪽 모터
const int IN2 = 6;
const int IN3 = 7;   // 오른쪽 모터
const int IN4 = 8;

const float STOP_DIST = 15.0;  // cm — 이보다 가까우면 정지

void setup() {
  pinMode(IR_L, INPUT);
  pinMode(IR_R, INPUT);
  pinMode(TRIG, OUTPUT);
  pinMode(ECHO, INPUT);
  pinMode(IN1, OUTPUT);
  pinMode(IN2, OUTPUT);
  pinMode(IN3, OUTPUT);
  pinMode(IN4, OUTPUT);
  Serial.begin(9600);
}

📌 무엇을 하는가

"어떤 핀에 무엇이 연결되어 있는지"를 코드에 먼저 알려주는 부분입니다. 이렇게 const로 이름을 붙여두면 나중에 핀 번호가 바뀌어도 한 줄만 고치면 돼서 편해요. 이번엔 충돌 방지용 TRIG/ECHO와 STOP_DIST(정지 거리 기준값)도 같이 정의합니다.

const int 바뀌지 않는 정수 상수
pinMode(핀, 모드) INPUT(읽기) / OUTPUT(쓰기)
💡
STOP_DIST를 const로 빼둔 이유: 나중에 더 멀리/가까이로 바꾸고 싶을 때 이 숫자 한 곳만 바꾸면 됩니다.

STEP 4-2 · 모터 동작 함수 4개

② 동작 함수 
void forward() {   // 직진
  digitalWrite(IN1, HIGH);
  digitalWrite(IN2, LOW);
  digitalWrite(IN3, HIGH);
  digitalWrite(IN4, LOW);
}
void turnLeft() {  // 좌회전: 왼쪽 정지
  digitalWrite(IN1, LOW);
  digitalWrite(IN2, LOW);
  digitalWrite(IN3, HIGH);
  digitalWrite(IN4, LOW);
}
void turnRight() { // 우회전: 오른쪽 정지
  digitalWrite(IN1, HIGH);
  digitalWrite(IN2, LOW);
  digitalWrite(IN3, LOW);
  digitalWrite(IN4, LOW);
}
void stopCar() {   // 모두 정지
  digitalWrite(IN1, LOW);
  digitalWrite(IN2, LOW);
  digitalWrite(IN3, LOW);
  digitalWrite(IN4, LOW);
}

📌 무엇을 하는가

차가 할 수 있는 4가지 동작(직진/좌회전/우회전/정지)을 각각 함수로 미리 만들어 두는 부분입니다. 이렇게 분리해 두면 메인 loop에서는 forward(); 한 줄만 부르면 됩니다.

🎯
핵심 아이디어: L298N은 모터마다 IN1·IN2 두 핀의 조합으로 방향을 결정합니다.
HIGH-LOW = 정방향 / LOW-HIGH = 역방향 / LOW-LOW = 정지.
↩️
조립 후 한쪽 바퀴가 거꾸로 돌면 코드를 고치지 말고 OUT 선 2개를 서로 바꿔 끼우세요.

STEP 4-3 · 거리 측정 함수 (초음파)

③ readDistance() 
float readDistance() {
  // 10μs 짧은 펄스로 초음파 발사
  digitalWrite(TRIG, LOW);
  delayMicroseconds(2);
  digitalWrite(TRIG, HIGH);
  delayMicroseconds(10);
  digitalWrite(TRIG, LOW);

  // 메아리가 돌아온 시간(μs) 측정 (timeout 30ms)
  long dur = pulseIn(ECHO, HIGH, 30000);

  // 시간 × 음속(0.034cm/μs) ÷ 2(왕복)
  return dur * 0.034 / 2;
}

📌 무엇을 하는가

앞쪽 장애물까지의 거리를 cm 단위로 반환하는 함수입니다. 초음파를 쏘고 → 돌아오는 시간을 재고 → 음속(약 340m/s)으로 거리를 계산하는 흐름이에요. 나누기 2를 하는 이유는 왕복 시간이라서 편도 거리만 필요하기 때문입니다.

pulseIn(핀, HIGH, timeout) HIGH 지속 시간(μs) 측정
⏱️
timeout 30000(30ms)이 중요한 이유: 메아리가 안 돌아오면 영원히 기다리게 됩니다. timeout을 걸어두면 측정 실패 시 0을 반환하고 다음 루프로 넘어가요.

STEP 4-4 · 메인 loop — 안전 체크 + 라인 추적

④ 메인 loop 
void loop() {
  // ── 1단계: 안전 우선 (충돌 방지) ──
  float dist = readDistance();
  if (dist > 0 && dist < STOP_DIST) {
    stopCar();
    Serial.print("⚠ 장애물 ");
    Serial.print(dist);
    Serial.println("cm — 정지");
    delay(100);
    return;     // 라인 추적 건너뛰기
  }

  // ── 2단계: 라인 추적 ──
  int L = digitalRead(IR_L);
  int R = digitalRead(IR_R);

  if (L == 0 && R == 0)
    forward();    // 직진
  else if (L == 1 && R == 0)
    turnLeft();   // 좌회전
  else if (L == 0 && R == 1)
    turnRight();  // 우회전
  else
    stopCar();    // 정지
}

📌 무엇을 하는가

loop는 끝없이 반복되면서 매번 두 단계를 거칩니다.
① 안전 체크: 앞 15cm 이내에 무언가 있으면 즉시 정지하고 라인 추적은 건너뜀.
② 라인 추적: 장애물이 없을 때만 평소처럼 IR 센서로 선을 따라감.

💬
이게 자율주행의 핵심 사고방식입니다. "주행"과 "안전"이 충돌하면 항상 안전이 이긴다는 우선순위. 진짜 자율주행차도 똑같이 작동해요 — 신호등이 초록불이어도 사람이 길에 있으면 멈추는 것처럼요.
🔑
return의 역할: 장애물이 있으면 return;으로 loop를 즉시 끝냅니다. 그래야 모터를 정지시킨 직후에 라인 추적 코드가 다시 모터를 켜지 않아요.

STEP 4-5 · 전체 코드 (모두 합치기)

위 네 덩어리를 그대로 이어 붙이면 충돌 방지가 적용된 라인트레이서 코드가 완성됩니다. 이걸 아두이노 IDE에 그대로 복사해서 업로드하세요.

LineTracer.ino — 완성품 (충돌 방지 포함) 
// ===== 자율주행 1단계: 라인트레이서 + 충돌 방지 =====
// IR 센서: 검은 선 = 1(HIGH), 흰 바닥 = 0(LOW)
// HC-SR04: 앞 STOP_DIST(15cm) 이내 장애물 감지 시 정지

const int IR_L = 2;
const int IR_R = 3;
const int TRIG = 9;
const int ECHO = 10;
const int IN1 = 5;   // 왼쪽 모터
const int IN2 = 6;
const int IN3 = 7;   // 오른쪽 모터
const int IN4 = 8;

const float STOP_DIST = 15.0;

void setup() {
  pinMode(IR_L, INPUT);
  pinMode(IR_R, INPUT);
  pinMode(TRIG, OUTPUT);
  pinMode(ECHO, INPUT);
  pinMode(IN1, OUTPUT); pinMode(IN2, OUTPUT);
  pinMode(IN3, OUTPUT); pinMode(IN4, OUTPUT);
  Serial.begin(9600);
}

// --- 모터 동작 함수 ---
void forward() {
  digitalWrite(IN1, HIGH); digitalWrite(IN2, LOW);
  digitalWrite(IN3, HIGH); digitalWrite(IN4, LOW);
}
void turnLeft() {
  digitalWrite(IN1, LOW);  digitalWrite(IN2, LOW);
  digitalWrite(IN3, HIGH); digitalWrite(IN4, LOW);
}
void turnRight() {
  digitalWrite(IN1, HIGH); digitalWrite(IN2, LOW);
  digitalWrite(IN3, LOW);  digitalWrite(IN4, LOW);
}
void stopCar() {
  digitalWrite(IN1, LOW); digitalWrite(IN2, LOW);
  digitalWrite(IN3, LOW); digitalWrite(IN4, LOW);
}

// --- 거리 측정 함수 ---
float readDistance() {
  digitalWrite(TRIG, LOW);  delayMicroseconds(2);
  digitalWrite(TRIG, HIGH); delayMicroseconds(10);
  digitalWrite(TRIG, LOW);
  long dur = pulseIn(ECHO, HIGH, 30000);
  return dur * 0.034 / 2;
}

void loop() {
  // 1) 안전 우선 — 장애물이 있으면 정지
  float dist = readDistance();
  if (dist > 0 && dist < STOP_DIST) {
    stopCar();
    Serial.print("⚠ 장애물 "); Serial.print(dist); Serial.println("cm");
    delay(100);
    return;
  }

  // 2) 라인 추적
  int L = digitalRead(IR_L);
  int R = digitalRead(IR_R);

  if (L == 0 && R == 0)       forward();
  else if (L == 1 && R == 0)  turnLeft();
  else if (L == 0 && R == 1)  turnRight();
  else                          stopCar();
}
